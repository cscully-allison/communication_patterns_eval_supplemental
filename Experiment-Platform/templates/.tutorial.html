In this study, you will be asked to interpret Gantt charts, a visualization commonly used to explore execution traces. We briefly describe the charts here.

We start with a timeline of a single processing element (e.g. MPI rank, CPU). The horizontal axis represents time. The colored boxes represent the state of the processing element. In the below example, the processing element is in the computation state first and then a communication state.

Instead of continous time, like the above example, we can also show activity in terms of discrete time steps. As shown below, the processing element is in a computation (gray) state at timesteps 1 and 3, the communication (blue) state at timestep 2. No boxes indicate the element is not participating in the timestep. In this example, the process does not participate in timestemps 4 through 7.

The remainder of this study will use time steps.

Gantt charts show multiple timelines in parallel. In the below example, PE1 and PE2 are in at communicaton state at timestep 2 but PE3 and P4 are not participating in that timestep.

When one processing element affects another, it is represented with a line. In the below example, PE1 sends a message to PE3 at timestep2 and PE3 receives the message at timestep 4.

When there are many processing elements, we may not be able to draw all of the timelines on screen. For the remainder of this study, we will draw only the first few timelines. For example, rather than drawing all the timelines of the chart on the left, we will instead draw the chart on the right.
